# ChatCoach - 微信聊天智能助手 需求文档

> 版本: v1.1
> 最后更新: 2026-03-03

---

## 一、产品概述

ChatCoach 是一款 Android 端微信聊天辅助工具。通过无障碍服务实时获取微信聊天内容，结合用户预设的身份关系与交流风格，调用大语言模型生成智能回复建议，以悬浮窗形式展示在微信聊天界面上方，帮助用户更得体、高效地进行日常沟通。

---

## 二、目标用户

- 社交沟通中希望获得措辞建议的用户
- 需要在职场、客户、长辈等不同场景切换语气的用户
- 希望提升聊天表达能力的用户

## 三、核心功能模块

### 3.1 好友管理模块

| 功能项 | 描述 |
|--------|------|
| 好友列表 | 展示已配置的微信好友，支持搜索/筛选 |
| 添加好友 | 手动输入好友备注名/微信名 |
| 好友画像设置 | 为每位好友配置以下信息：|
| - 身份关系 | 预设标签（同事/领导/客户/朋友/恋人/家人/长辈）+ 自定义 |
| - 交流语气 | 预设选项（正式/轻松/亲切/幽默/尊敬/专业）+ 自定义 |
| - 沟通态度 | 预设选项（主动热情/礼貌克制/随和自然/谨慎委婉）+ 自定义 |
| - 自定义 Prompt | 用户可编写完整的自定义提示词，精细控制 AI 回复风格 |
| - 备注信息 | 关于该好友的补充说明（如性格特点、禁忌话题等） |
| 好友编辑/删除 | 修改或移除已配置好友 |

### 3.2 聊天监控模块（无障碍服务）

| 功能项 | 描述 |
|--------|------|
| 无障碍服务 | 通过 AccessibilityService 实时监听微信聊天界面 |
| 聊天内容提取 | 解析当前聊天窗口中的消息列表（发送者、内容、时间） |
| 好友识别 | 自动识别当前聊天对象的名称，匹配已配置好友 |
| 新消息检测 | 检测到新消息时自动触发回复建议生成 |
| 消息缓存 | 缓存最近 N 条消息作为上下文（默认 20 条，可配置） |
| 语音消息识别 | 检测到语音消息时，提取微信界面上"转文字"后的文本内容，纳入聊天上下文 |
| 服务状态指示 | 在应用内显示无障碍服务是否正常运行 |

### 3.3 大模型接入模块

| 功能项 | 描述 |
|--------|------|
| 预设平台 | 内置以下平台的接口配置模板：|
| - OpenAI | ChatGPT 系列（gpt-4o、gpt-4o-mini 等） |
| - Claude | Anthropic Claude 系列 |
| - DeepSeek | DeepSeek-Chat / DeepSeek-Reasoner |
| - Gemini | Google Gemini 系列（gemini-pro、gemini-1.5-pro 等） |
| - 通义千问 | 阿里云通义千问系列 |
| - 本地模型 | Ollama 本地部署模型 |
| 自定义接入 | 用户可手动填写：API URL、API Key、模型名称、请求格式 |
| 接口测试 | 配置完成后可一键测试连通性 |
| 多模型切换 | 支持配置多个模型，可为不同好友指定不同模型 |
| Token 用量统计 | 统计每次调用的 Token 消耗量 |

### 3.4 智能回复模块

| 功能项 | 描述 |
|--------|------|
| 回复生成 | 将聊天上下文 + 好友画像 Prompt 发送给 LLM，获取回复建议 |
| 多风格回复 | 每次生成 3 条不同风格的回复建议，每条附带风格标签 |
| 风格标签 | 每条回复建议显示风格标签（如：正式/幽默/暖心/简洁/俏皮/专业），标签以彩色 Chip 形式展示在回复卡片顶部 |
| 一键复制 | 点击回复建议即复制到剪贴板，方便粘贴到微信输入框 |
| 重新生成 | 对当前建议不满意时可重新生成 |
| 手动触发 | 除自动触发外，支持手动点击按钮请求生成建议 |
| 上下文管理 | 自动裁剪过长的上下文以控制 Token 消耗 |
| 长记录摘要压缩 | 当聊天记录超过阈值（可配置，默认 30 条）时，先将早期消息通过 LLM 生成摘要，再将摘要 + 最近消息拼接作为上下文发送，显著节省 Token |
| 流式输出 | 支持 SSE 流式输出，逐字显示回复（提升体验） |

### 3.5 悬浮窗模块

| 功能项 | 描述 |
|--------|------|
| 悬浮窗展示 | 半透明悬浮窗覆盖在微信界面上方 |
| 折叠/展开 | 默认折叠为小图标，点击展开显示回复建议卡片 |
| 拖拽移动 | 悬浮窗可自由拖拽到屏幕任意位置 |
| 自动隐藏 | 离开微信界面时自动隐藏，回到微信时自动显示 |
| 透明度调节 | 支持调节悬浮窗透明度（设置中配置） |
| 回复卡片 | 展开后显示多条回复建议，每条可复制/收藏 |
| 加载动画 | 等待 LLM 响应时显示加载动画 |

### 3.6 聊天记录分析模块

| 功能项 | 描述 |
|--------|------|
| 风格分析 | 分析好友的聊天风格（用词习惯、句式偏好） |
| 情绪倾向 | 识别好友当前的情绪状态（积极/消极/中性） |
| 话题偏好 | 统计好友常聊话题，提供话题建议 |
| 分析报告 | 生成好友聊天分析报告（可在好友详情页查看） |

### 3.7 聊天复盘模块

| 功能项 | 描述 |
|--------|------|
| 对话复盘 | 对一段已结束的完整对话进行事后分析，回顾整个沟通过程 |
| 沟通评分 | 从表达清晰度、语气得体性、情绪管理、话题推进 4 个维度评分（1-10 分） |
| 亮点与不足 | 标注对话中表达得体的消息（亮点）和可以改进的消息（不足） |
| 改进建议 | 针对不足之处给出具体的改写建议，展示"原句 → 建议句"对比 |
| 沟通策略建议 | 基于整段对话总结后续与该好友沟通的策略建议 |
| 复盘历史 | 保存历次复盘记录，可回顾查看进步轨迹 |
| 手动触发 | 用户在好友详情页选择一段对话时间范围后手动发起复盘 |

### 3.8 快捷回复模板模块

| 功能项 | 描述 |
|--------|------|
| 预设模板 | 内置常见场景模板：问候、感谢、道歉、拒绝、祝福、约定等 |
| 自定义模板 | 用户可创建自定义快捷回复模板 |
| 模板分类 | 按场景分类管理模板 |
| 智能推荐 | 根据当前聊天上下文推荐可能适用的模板 |
| 一键使用 | 点击模板直接复制，或基于模板生成个性化回复 |

## 四、系统 Prompt 构建策略

系统 Prompt 由以下部分自动拼接：

### 4.1 实时回复建议 Prompt

```
[系统角色设定]
你是一位社交沟通助手，帮助用户在微信聊天中给出合适的回复建议。

[好友画像]
- 我与对方的关系：{身份关系}
- 我应该使用的语气：{交流语气}
- 我的沟通态度：{沟通态度}
- 补充说明：{备注信息}

[用户自定义 Prompt]（如有）
{自定义 Prompt 内容}

[输出要求]
请根据以下聊天记录，生成 3 条不同风格的回复建议。
每条回复需包含：
- 风格标签（如：正式/幽默/暖心/简洁/俏皮/专业，选择最贴切的一个）
- 回复内容（控制在 50 字以内）

输出格式：
【风格标签】回复内容

直接给出回复内容，不要解释。

[聊天记录]
{最近 N 条消息，或：摘要 + 最近消息}
```

### 4.2 长记录摘要压缩策略

当聊天记录条数超过阈值时，上下文构建流程为：

```
原始消息列表（如 50 条）
    ↓
拆分为：早期消息（前 30 条）+ 近期消息（后 20 条）
    ↓
早期消息 → 发送给 LLM 生成摘要（约 200 字）
    ↓
最终上下文 = [摘要段落] + [近期 20 条消息原文]
```

摘要 Prompt：
```
请将以下聊天记录压缩为一段简要摘要（200字以内），
保留关键话题、双方立场和情绪变化，省略寒暄和重复内容：
{早期消息}
```

### 4.3 聊天复盘 Prompt 策略

```
[系统角色设定]
你是一位沟通教练，擅长分析人际对话并给出改进建议。

[好友画像]
- 我与对方的关系：{身份关系}
- 我期望的语气：{交流语气}
- 我的沟通态度：{沟通态度}

[分析要求]
请对以下完整对话进行复盘分析，输出以下内容：

1. 沟通评分（1-10分）：
   - 表达清晰度
   - 语气得体性
   - 情绪管理
   - 话题推进

2. 亮点消息（标注序号和原因）

3. 可改进消息（标注序号 + 原句 → 建议改写）

4. 后续沟通策略建议（2-3 条）

[完整对话记录]
{完整对话}
```

## 五、UI/UX 设计规范

### 5.1 设计风格

- **设计语言**: Material Design 3 (Material You)
- **配色方案**: 以微信绿（#07C160）为主色调，搭配温暖中性色
- **整体风格**: 清新、温暖、亲切，避免冰冷的科技感
- **圆角设计**: 大圆角卡片（16dp），按钮圆角（12dp）
- **图标风格**: 圆润线性图标，2dp 线宽
- **动效**: 平滑过渡动画，弹性交互反馈

### 5.2 主要页面

| 页面 | 描述 |
|------|------|
| 启动页 | 品牌 Logo + 应用名称，简短加载 |
| 首页/仪表盘 | 服务状态卡片、快捷操作入口、最近活跃好友 |
| 好友列表页 | 已配置好友的卡片列表，显示关系标签和在线状态 |
| 好友详情/编辑页 | 好友画像配置表单，分步设置 |
| 模型设置页 | LLM 平台选择与配置 |
| 快捷模板页 | 模板分类浏览与管理 |
| 聊天分析页 | 好友聊天风格分析报告 |
| 聊天复盘页 | 选择对话时间范围，查看复盘分析结果（评分/亮点/改进/策略） |
| 设置页 | 通用设置（悬浮窗、消息条数、摘要压缩阈值、Token 统计等） |

### 5.3 悬浮窗设计

- **折叠态**: 48dp 圆形图标，带呼吸灯效果（有新建议时）
- **展开态**: 宽度为屏幕宽度 85%，最大高度 40% 屏幕高度
- **卡片样式**: 每条回复建议为独立卡片，顶部显示彩色风格标签 Chip（如 正式/幽默/暖心），左滑可收藏，点击复制
- **顶部信息**: 当前好友名称 + 匹配状态
- **底部操作**: 重新生成按钮 + 手动触发按钮 + 设置按钮

## 六、技术架构

### 6.1 整体架构

```
┌─────────────────────────────────────────────┐
│                  UI Layer                    │
│  Activities / Fragments / Floating Window    │
│            (Material Design 3)               │
├─────────────────────────────────────────────┤
│               ViewModel Layer                │
│        (Jetpack ViewModel + LiveData)        │
├─────────────────────────────────────────────┤
│              Repository Layer                │
│     (Data coordination & business logic)     │
├──────────────┬──────────────┬───────────────┤
│  Local Data  │  LLM Service │ Accessibility │
│   (Room DB)  │  (Retrofit)  │   Service     │
└──────────────┴──────────────┴───────────────┘
```

### 6.2 技术栈

| 类别 | 技术选型 |
|------|----------|
| 语言 | Kotlin |
| UI 框架 | Android Views + Material Design 3 |
| 架构模式 | MVVM (Model-View-ViewModel) |
| 数据库 | Room (SQLite) |
| 网络请求 | Retrofit2 + OkHttp3 |
| 异步处理 | Kotlin Coroutines + Flow |
| 依赖注入 | 手动依赖注入（轻量级，避免引入 Hilt/Dagger 增加复杂度） |
| 悬浮窗 | WindowManager + System Alert Window |
| 聊天监控 | AccessibilityService |
| JSON 解析 | Gson |
| 图片加载 | 无（本应用不涉及网络图片） |
| 导航 | Jetpack Navigation |
| 数据存储 | SharedPreferences（轻量配置）+ Room（结构化数据） |

## 八、非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | LLM 响应超时设置为 30 秒，支持流式输出减少等待感 |
| 隐私安全 | API Key 使用 Android Keystore 加密存储；聊天记录仅本地缓存，不上传至任何第三方服务器（仅发送给用户配置的 LLM） |
| 电量优化 | 无障碍服务仅在微信前台时活跃监听，其余时间低功耗待机 |
| 兼容性 | 最低支持 Android 7.0 (API 24)，目标 API 36 |
| 存储 | 聊天缓存自动清理（保留最近 7 天），可手动清空 |
| 容错 | 网络异常/LLM 调用失败时友好提示，不影响微信正常使用 |
| 无障碍服务稳定性 | 服务断开时自动提醒用户重新启用 |
